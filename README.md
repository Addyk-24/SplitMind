# SplitMind

SplitMind is a Codex Skill and CLI orchestration layer for safer agentic coding.

Instead of asking one AI agent to guess one implementation path, SplitMind runs multiple coding strategies in parallel Git worktrees, evaluates each branch with real tests, and merges only the safest passing result.

> Codex should not guess once. Codex should explore, test, and converge.

## Why It Matters

Single-path AI coding is fragile. If the agent chooses the wrong architecture, it can hallucinate, over-refactor, or quietly break behavior.

SplitMind treats coding as a tournament:

1. Create isolated Git worktrees.
2. Run different agent strategies in parallel.
3. Execute deterministic tests in every branch.
4. Select the best passing candidate.
5. Merge the winner into `main`.
6. Prune failed or non-winning branches.

This brings ideas from beam search, tournament selection, and CI-driven validation into Codex-style software work.

## CLI

SplitMind is controlled from the CLI. The web dashboard is optional telemetry; the CLI is the product surface that owns worktrees, tests, winner selection, and merge policy.

Initialize local config:

```bash
python splitmind.py init
```

Run a swarm:

```bash
python splitmind.py run "Fix the corrupted payment gateway implementation and preserve all payment contract tests."
```

Prepared demo shortcut:

```bash
python splitmind.py demo
```

Inspect the latest run:

```bash
python splitmind.py status
python splitmind.py results
python splitmind.py results sm-tdd
```

Check the local machine:

```bash
python splitmind.py doctor
```

Clean stale worktrees:

```bash
python splitmind.py clean
```

Use this task during judging:

```text
Fix the corrupted payment gateway implementation and preserve all payment contract tests.
```

Low-level orchestrator command:

```bash
python backend/splitmind_orchestrator.py \
  --task "Fix the IN locale currency bug without breaking cart total and discount tests" \
  --strategies "Locale Validation Layer|Guard Clause Fix|Contract-First Rewrite" \
  --target-file "src/payment_gateway.py" \
  --test-command "pytest tests/ -v --tb=short --no-header" \
  --results-dir "." \
  --base-branch "main"
```

## Architecture

```text
User task
  -> SplitMind Skill
  -> Strategy planner
  -> Git worktree A / B / C
  -> Parallel coding agents
  -> Production test command
  -> Winner selection
  -> Merge to main
  -> Prune non-winning worktrees
```

## Dashboard

The Next.js dashboard is optional telemetry, not the core product. It lets judges and developers inspect the autonomous loop:

- live branch and agent status
- winner vs tiebreak-pruned vs failed branches
- before/after code generated by each agent
- changed lines
- test output and exit code
- final merge decision

Run it from `splitmind-frontend`:

```bash
npm run dev
```

Then open `http://localhost:3000`.

## Agent Adapters

SplitMind can be exposed to different agentic coding tools as a thin instruction layer over the same CLI:

- Codex Skill: `my-skill/SKILL.md` or `adapters/codex/SKILL.md`
- Claude project command/instruction: `adapters/claude/splitmind.md`
- Cursor rule: `adapters/cursor/splitmind.mdc`

All adapters route complex coding tasks back to:

```bash
python splitmind.py run "<task>"
```

This keeps the merge policy, test validation, and worktree cleanup centralized in the CLI instead of duplicating behavior across editors.

## Current Prototype Scope

The hackathon prototype demonstrates the full loop on a controlled payment bug scenario. The CLI already exposes the important generalization points:

- `python splitmind.py run`
- `python splitmind.py init`
- `python splitmind.py status`
- `python splitmind.py results`
- `python splitmind.py doctor`
- `python splitmind.py clean`
- `.splitmind.json`
- `--strategies`
- `--target-file`
- `--test-command`
- `--results-dir`
- `--base-branch`

The next step is broader repository support: multiple changed files, configurable merge policies, and deeper Codex worker integration.
